新機能は Preconnect オプションです。「ページの最適化」->「[3] HTML の設定」の中にあります。

rel=preconnect - HTML: HyperText Markup Language | MDN

特定のサイトを先読みする機能のようです。私には使い道がなさそうですので空白のままにしておきます。

新機能のもう一つは Woocommerce 向けのようですので省略します。

バグフィックスの内容を見ると脆弱性への対策 (vulnerability issue) も含まれるようです。早めのアップデートをお勧めします。

WordPress とプラグインは出来るだけ最新に

広く使われている WordPress と、そのプラグインは、セキュリティーホールが見つかった場合に影響が大きいと思います。アップデートして不具合が発生した場合は別として、出来るだけ最新に維持する方が良いかと思います。

これからも、人柱半分、興味半分で、出来るだけ早めに更新していきたいと考えています。

LiteSpeed Cache プラグインの最適化方法

これまで取り組んできた LiteSpeed Cache プラグインの最適化についてまとめたページです。興味のある方は参照下さい。PageSpeed Insights のスコアは、Desktop / Mobile 共に 90 以上を維持できます。

LOLIPOP! で LiteSpeed Cache を使ってみました。導入後に QUIC.cloud へ接続する所まで設定しました。CDN への接続は DNS サーバの設定変更が必要なので、この部分は後回しにしておきます。

QUIC.cloud の DNS を使用して LiteSpeed Cache で CDN を有効にする方法
QUIC.cloud の DNS サーバーを使用することで、LiteSpeed Cache プラグインで CDN を有効にする事が出来ました。ドメインの管理をロリポップ！からムームードメインに移行し、更に DNS サーバーを QUIC.cloud に変更すれば、CDN を使用するのに必要な FQDN を CNAME で追加する事が出来ます。

Cocoon 2.6.8 アップデート

WordPress の Cocoon テーマが Ver.2.6.8 にアップデートされました。リストブロックの装飾とスタイルの装飾、twitter -> X が主な変更箇所です。新しい装飾が表示されない場合にはキャッシュが悪影響しています。

独自ドメインの管理を LOLIPOP! からムームードメインに移行する

DNS レコードの編集が LOLIPOP! では出来ないので、独自ドメインの管理をムームードメインに変更しました。サブドメインの追加程度なら LOLIPOP! でもかまわないと思いますが、これから先の事を考えて移行しました。

Cocoon 2.6.1.2 アップデート

WordPress の Cocoon テーマが 2.6.1.2 にアップデートされました。段落ブロックのボーダー・スタイル追加と、AMP・PWA の非推奨 (将来的な廃止も) が大きな変更箇所です。特に不具合なく更新できました。

Cocoon 2.6.6 アップデート

WordPress の Cocoon テーマが Ver.2.6.6 にアップデートされました。主にステマ規制に対応するための PR 表記機能の追加です。テーブルの1列目が透けて文字が重なる不具合が解消されました。

複数ページの Lighthouse の結果を一括で取得する
