MAIS DE 17.000 SITES WORDPRESS COMPROMETIDOS PELO BALADA INJECTOR EM SETEMBRO DE 2023

byCanalFs0ciety -11:00 0 Comentários

 

Mais de 17.000 sites WordPress foram comprometidos no mês de setembro de 2023 com malware conhecido como Balada Injector, quase o dobro do número de detecções em agosto.

Destes, 9.000 sites teriam sido infiltrados usando uma falha de segurança recentemente divulgada no plug-in tagDiv Composer (CVE-2023-3169, pontuação CVSS: 6.1) que poderia ser explorada por usuários não autenticados para executar scripts entre sites armazenados. (XSS) ataques.

“Esta não é a primeira vez que a gangue Balada Injector tem como alvo vulnerabilidades nos temas premium do tagDiv”, disse o pesquisador de segurança da Sucuri, Denis Sinegubko.

“Uma das primeiras injeções massivas de malware que poderíamos atribuir a esta campanha ocorreu durante o verão de 2017, onde bugs de segurança divulgados em temas WordPress de jornais e notícias foram ativamente abusados.”

Balada Injector é uma operação em grande escala descoberta pela primeira vez pelo Doctor Web em dezembro de 2022, na qual os agentes da ameaça exploram uma variedade de falhas de plug-ins do WordPress para implantar um backdoor do Linux em sistemas suscetíveis.

O principal objetivo do implante é direcionar os usuários dos sites comprometidos para páginas falsas de suporte técnico, ganhos fraudulentos em loterias e golpes de notificação push. Mais de um milhão de sites foram impactados pela campanha desde 2017.

Os ataques envolvendo o Balada Injector ocorrem na forma de ondas de atividade recorrentes que ocorrem a cada duas semanas, com um aumento nas infecções detectadas às terças-feiras após o início de uma onda durante o fim de semana.

O último conjunto de violações envolve a exploração do CVE-2023-3169 para injetar um script malicioso e, em última análise, estabelecer acesso persistente aos sites, carregando backdoors, adicionando plug-ins maliciosos e criando administradores de blogs desonestos.

Historicamente, esses scripts têm como alvo administradores de sites WordPress logados, pois permitem que o adversário execute ações maliciosas com privilégios elevados por meio da interface administrativa, incluindo a criação de novos usuários administradores que podem usar para ataques subsequentes.

A natureza de rápida evolução dos scripts é evidenciada por sua capacidade de implantar um backdoor nas páginas de erro 404 dos sites que são capazes de executar código PHP arbitrário ou, alternativamente, aproveitar o código incorporado nas páginas para instalar um plugin wp-zexit malicioso. de forma automatizada.

A Sucuri o descreveu como “um dos tipos de ataques mais complexos” executados pelo script, já que ele imita todo o processo de instalação de um plugin a partir de um arquivo ZIP e sua ativação.

A funcionalidade principal do plugin é a mesma do backdoor, que consiste em executar código PHP enviado remotamente pelos agentes da ameaça.

Ondas de ataques mais recentes observadas no final de setembro de 2023 envolvem o uso de injeções de código aleatório para baixar e lançar um malware de segundo estágio de um servidor remoto para instalar o plugin wp-zexit.

Também são usados ​​scripts ofuscados que transmitem os cookies do visitante para uma URL controlada pelo ator e buscam em troca um código JavaScript não especificado.

“A colocação deles nos arquivos dos sites comprometidos mostra claramente que desta vez, em vez de usar a vulnerabilidade do tagDiv Composer, os invasores aproveitaram seus backdoors e usuários administradores mal-intencionados que foram plantados após ataques bem-sucedidos contra administradores de sites”, explicou Sinegubko.
